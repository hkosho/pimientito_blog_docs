# Webサーバ公開 基盤設定手順書

## 📖 このドキュメントについて

Ubuntu Server 24.04 LTS上で、セキュアなWebサーバを構築するための手順書です。
実際のサーバ構築プロジェクトで使用した手順を基に、生成AI（ChatGPT、Claude、Gemini）との協働により作成され、実環境での検証済みです。

**手順書本体**: [WebServer_Infra_Setup.md](./WebServer_Infra_Setup.md)

---

## 🎯 対象読者

- **Linux初心者〜中級者**
  - コマンドライン操作の基礎知識があれば実行可能
  - サーバ構築が初めての方でも、手順に従えば完了できる設計
  - Windows環境からのリモート構築を想定

## 💻 想定環境

| 項目 | 内容 |
|------|------|
| **サーバOS** | Ubuntu Server 24.04 LTS |
| **ネットワーク** | LAN環境下での構築 |

### 実装時の注意
- `[username]`、`[your-domain.com]`、`[myproject]`、`XXX.XXX.XXX.XXX` 等のプレースホルダーは、**必ず実際の環境に合わせて置換してください**

- サンプル値をそのまま使用しないでください
- 本番環境での実施前に、テスト環境での検証を強く推奨します

## ✅ この手順書でできること

### 1. セキュアなSSH接続環境の構築
- **クライアント側**での鍵ペア生成（セキュリティベストプラクティス）
- 鍵認証によるSSH接続（パスワードレス）
- SSHポート変更とパスワード認証無効化
- TeraTerm自動ログインスクリプト作成

### 2. Webサーバの基盤構築
- nginxのインストールと仮想ホスト設定
- 適切なディレクトリ構造の構築

### 3. セキュリティ対策
- ファイアウォール（ufw）設定
- fail2banによる侵入防止（自己BAN対策含む）
- 適切なファイル権限管理（最小権限の原則）

### 4. 運用基盤の整備
- OS自動更新の設定（セキュリティパッチ自動適用）
- 定期バックアップの自動化（cron）
- ログローテーション設定
- HTTPS対応（Let's Encrypt）
- 外部バックアップの重要性と実施方法

### 5. 運用ガイドライン
- 定期メンテナンススケジュール例
- 完了チェックリスト
- 最終確認手順
- トラブルシューティング（ロックアウト救出策含む）

## ⚠️ 重要な注意事項

### セキュリティについて
- 本手順書のプレースホルダーは、**必ず実際の環境に合わせて置換してください**
- 特に以下は推測困難な値を設定：
  - ユーザ名
  - SSHポート番号
  - ドメイン名
- サンプル値（`webuser`、`22222`等）をそのまま使用しないでください
- 本番環境での実施前に、テスト環境での検証を推奨します

### 構築手順について
- 手順の順序は重要です（特にSSH設定・ファイアウォール設定）
- ロックアウトを防ぐため、SSH設定変更は慎重に実施してください
- 既存セッションを維持したまま新ポートでの接続確認を行ってください
